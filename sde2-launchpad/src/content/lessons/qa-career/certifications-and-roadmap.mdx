Certifications open doors and structure your learning, but **skills + a public portfolio get you hired**. Treat certs as supporting evidence.

## The certifications worth considering

- **ISTQB Foundation Level (CTFL)** — the well-known starting point. Covers principles, test design, levels/types, defect lifecycle. Great for vocabulary and for clearing HR filters in many regions.
- **ISTQB CTAL-TAE (Test Automation Engineer)** — the advanced cert aimed squarely at automation/SDET roles. The natural next step after Foundation.
- **Supporting**: a cloud cert (**AWS Cloud Practitioner**) for CI/cloud context, an **Agile/Scrum** cert, and vendor/course completions (Selenium, Playwright via Test Automation University — free).

## QA Engineer vs SDET

- **QA / Test Engineer** — focuses on test design, manual + some automation, quality process.
- **SDET (Software Development Engineer in Test)** — a developer who specializes in testing: builds frameworks and tooling, automates across UI/API/unit, owns CI/CD for tests, and codes daily. SDET roles pay more and have grown ~50% since 2024.

## The realistic roadmap

1. **Manual tester** — learn fundamentals + test design (Foundations track).
2. **Automation tester** — one language + Selenium/Playwright + a framework + API tests.
3. **SDET** — add CI/CD, Docker, performance, AI tooling; build tools, own quality infrastructure.
4. **Lead / Architect** — strategy, mentoring, test architecture across teams.
