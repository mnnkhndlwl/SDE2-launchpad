The core idea: a **hash map / set gives O(1) average lookup**, so instead of re-scanning the array to answer "have I seen this?" or "how many of these?", you answer in constant time. You trade O(n) space for a huge time win.

## The three sub-patterns

**1. Seen-set / complement lookup.** As you scan, store what you've seen; check if the thing you need is already there.

```js
// Two Sum: for each x, is (target - x) already seen?
function twoSum(nums, target) {
  const seen = new Map();
  for (let i = 0; i < nums.length; i++) {
    const need = target - nums[i];
    if (seen.has(need)) return [seen.get(need), i];
    seen.set(nums[i], i);
  }
}
```

**2. Frequency counting.** Tally occurrences in a map, then reason over counts (anagrams, top-K, majority).

```js
const freq = new Map();
for (const c of s) freq.set(c, (freq.get(c) || 0) + 1);
```

**3. Prefix products / sums.** Precompute running aggregates so each answer is O(1). Product Except Self multiplies a left-prefix pass by a right-suffix pass — no division, O(n).

<div class="callout warn">
  <div class="ct">The O(1) caveat</div>
  Hash lookups are O(1) <i>average</i>, not worst-case. With adversarial keys and collisions they degrade to O(n). For interviews, state "O(1) average" — it shows you know the difference.
</div>
