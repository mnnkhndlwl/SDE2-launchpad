A binary tree is recursion made visible: almost every tree problem is "do something at this node, then recurse on the children." Four traversal orders cover nearly everything.

## See the four orders

<TreeTraversal />

- **Preorder** (root → left → right): copy/serialize a tree, process parent before children.
- **Inorder** (left → root → right): on a **BST this yields sorted order** — a hugely useful fact.
- **Postorder** (left → right → root): process children before parent — deleting a tree, computing heights/sizes bottom-up.
- **BFS / level-order**: a **queue**, level by level — shortest paths in unweighted graphs, level grouping.

## DFS is recursion; BFS is a queue

```js
// DFS (preorder)
function dfs(node, out = []) {
  if (!node) return out;
  out.push(node.val);
  dfs(node.left, out);
  dfs(node.right, out);
  return out;
}

// BFS (level order)
function bfs(root) {
  const out = [], q = root ? [root] : [];
  while (q.length) {
    const level = [];
    for (let n = q.length; n > 0; n--) {  // snapshot this level's size
      const node = q.shift();
      level.push(node.val);
      if (node.left) q.push(node.left);
      if (node.right) q.push(node.right);
    }
    out.push(level);
  }
  return out;
}
```

## The BST validation trap

A common bug: checking only `left.val < node.val < right.val` locally. That's wrong — every node in the left **subtree** must be smaller, not just the direct child. Carry down valid bounds:

```js
function isValidBST(node, lo = -Infinity, hi = Infinity) {
  if (!node) return true;
  if (node.val <= lo || node.val >= hi) return false;
  return isValidBST(node.left, lo, node.val) &&
         isValidBST(node.right, node.val, hi);
}
```
