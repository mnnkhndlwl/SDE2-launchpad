Playwright is a modern automation framework from Microsoft. One API drives Chromium, Firefox and WebKit, in JS/TS, Python, Java and C#.

## Why it's winning

- **Auto-waiting** ‚Äî every action waits for the element to be actionable. No manual waits, far less flakiness.
- **Resilient locators** ‚Äî `getByRole`, `getByText`, `getByLabel`, `getByTestId` target what the *user* sees.
- **Web-first assertions** ‚Äî `expect(locator).toBeVisible()` retries until true or times out.
- **Fast parallelism** out of the box, plus the killer **trace viewer** for debugging.

## First test

```ts
import { test, expect } from '@playwright/test';

test('checkout flow', async ({ page }) => {
  await page.goto('https://www.saucedemo.com/');
  await page.getByPlaceholder('Username').fill('standard_user');
  await page.getByPlaceholder('Password').fill('secret_sauce');
  await page.getByRole('button', { name: 'Login' }).click();
  await expect(page.getByText('Products')).toBeVisible();
});
```

No explicit waits ‚Äî Playwright auto-waits for each step.

## Network mocking

```ts
await page.route('**/api/cart', route =>
  route.fulfill({ json: { items: [] } }));
```

Stub back-end responses to test edge cases (errors, empty states) deterministically.

## Reuse a logged-in session (huge speed win)

```ts
// global setup: log in once, save storage state
await page.context().storageState({ path: 'auth.json' });
// then in config: use: { storageState: 'auth.json' }
```

Every test starts already authenticated ‚Äî no re-login per test.

## Debugging superpowers

- `npx playwright test --ui` ‚Äî interactive UI mode.
- `npx playwright codegen` ‚Äî record actions into code.
- **Trace viewer** ‚Äî time-travel through a failed run with DOM snapshots, network and console.
