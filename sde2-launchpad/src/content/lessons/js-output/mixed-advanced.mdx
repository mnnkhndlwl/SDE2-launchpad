<details>
<summary><strong>Answer & why</strong></summary>


Object key ordering: **integer-like keys first** (ascending numeric), then string keys in **insertion order**. `'0'` and `'1'` come before `'b'`, `'a'`, `'c'`. This is a spec-guaranteed order since ES2015.

## Q6 â€” generator yield and return

<JsPlayground code={`function* gen() {

console.log(g.next()); // ?

<details>
<summary><strong>Answer & why</strong></summary>

```

```
`yield` produces `{ value, done: false }`. `return` produces `{ value, done: true }`. Once `done: true`, all subsequent `next()` calls return `{ value: undefined, done: true }`. `yield 4` is unreachable code.
</details>

## Q7 â€” Promise.race and Promise.any

<JsPlayground code={`const p1 = new Promise(r => setTimeout(() => r('slow'), 200));

<details>
<summary><strong>Answer & why</strong></summary>

```

```

</details>

## Q11 â€” for...in vs for...of

<JsPlayground code={`const arr = [10, 20, 30];
arr.custom = 'extra';

for (const k in arr) {
  process.stdout ? process.stdout.write(k + ' ') : null;
}
console.log('(for..in)');

for (const v of arr) {
  process.stdout ? process.stdout.write(v + ' ') : null;
}
console.log('(for..of)');`} height={175} />

<details>
<summary><strong>Answer & why</strong></summary>

```
0 1 2 custom (for..in)
10 20 30 (for..of)
```
`for...in` iterates **all enumerable property keys** (including inherited ones and added properties like `custom`). `for...of` uses the **iterator protocol** â€” for arrays this iterates **values** only, ignoring non-numeric properties. Never use `for...in` on arrays.
</details>
