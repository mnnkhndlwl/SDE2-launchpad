Appium automates native, hybrid and mobile-web apps on Android and iOS using the same WebDriver protocol you already know — so your Selenium skills transfer.

## App types

- **Native** — built with platform SDKs (Kotlin/Swift). Fastest, full device access.
- **Hybrid** — web content in a native shell; you switch between native and web *contexts*.
- **Mobile web** — a site in the mobile browser; test like web but on mobile viewports.

## Emulator/simulator vs real device

- **Emulators** — cheap, fast, parallelizable, good for early dev.
- **Real devices** — catch performance, sensors, network and OEM quirks. Use cloud farms (BrowserStack/Sauce App Automate) for breadth.

## Core concepts

- **Desired capabilities** — platform, device, app path/package, automation engine (UiAutomator2 for Android, XCUITest for iOS).
- **Appium Inspector** — point at the app to discover locators (accessibility id, id, XPath).
- **Contexts** — `NATIVE_APP` vs `WEBVIEW_*` for hybrid apps.
- **Gestures** — tap, swipe, scroll, long-press via the W3C Actions API.

## Modern alternatives

**Maestro** (simple YAML flows), and platform-native **Espresso** (Android) / **XCUITest** (iOS) for dev-owned tests. Appium remains the cross-platform standard.
