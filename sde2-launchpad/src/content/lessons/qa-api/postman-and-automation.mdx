Start exploratory in **Postman**, then move stable checks into **code** that runs in CI.

## Postman essentials

- **Collections** group related requests; **environments** hold variables (base URL, token).
- Write tests in the *Tests* tab:

```js
pm.test("status is 200", () => pm.response.to.have.status(200));
pm.test("has a token", () => pm.expect(pm.response.json().token).to.exist);
pm.environment.set("token", pm.response.json().token); // chain to next request
```

- Run the whole collection headlessly in CI with **Newman** (`newman run collection.json`).

## Automate in code — REST Assured (Java)

```java
given()
    .header("Content-Type", "application/json")
    .body("{ \"email\": \"a@b.com\", \"password\": \"x\" }")
.when()
    .post("https://reqres.in/api/login")
.then()
    .statusCode(200)
    .body("token", notNullValue())
    .time(lessThan(2000L));
```

## Automate in code — requests + pytest (Python)

```python
import requests

def test_create_user():
    r = requests.post("https://reqres.in/api/users",
                      json={"name": "sahil", "job": "sdet"})
    assert r.status_code == 201
    body = r.json()
    assert body["name"] == "sahil"
    assert "id" in body
```

## Schema validation & chaining

- Validate responses against a **JSON schema** so structural changes fail loudly.
- **Chain**: capture a token/id from one response and pass it to the next request (auth flows, create-then-fetch).
- **Mock** unstable dependencies with WireMock / Mockoon so tests stay deterministic.
